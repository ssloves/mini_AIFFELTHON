> (개선 로드맵 1~7)** 과정에서 얻은 통찰을 반영해 재설계한 **최종 실험계획서**다.

## 1. 실험 주제

한국 노동·세법 5개 문서(고용보험법시행령·근로기준법시행령·조세특례제한법·조세특례제한법시행령·
청년일자리도약장려금 지침)로 구성한 지식 그래프 기반 **GraphRAG 에이전트**가, 동일한 청크·임베딩을
쓰는 **순수 벡터 RAG(Naive)** 대비 **근거 조항 수집·추론 경로·답변 신뢰성에서 우위를 갖는지**를,
요인을 하나씩 더하는 **5단 ablation 사다리(B0~B4)** 로 분해 검증한다.

핵심 가설(불변):
> **"GraphRAG는 더 똑똑한 답이 아니라, 더 신뢰할 수 있는(근거를 안 놓치고·올바른 구조로 도달하는) 답을 낸다."**
> → 1순위 지표는 Answer Accuracy가 아니라 **Context Recall**과 **경로 충실도(Path Faithfulness)**.

### 1.1 1차 실험에서 얻은 통찰 → 계획 재설계 근거

| 1차에서 드러난 문제 | 강화 조치(로드맵) | 계획 반영 |
|----|----|----|
| 충돌 문항 Recall 우위 일부가 "설계상 정준쌍 주입(by-construction)" | 출처(held_out/generated) 분리·정밀 채점 | 출처별 분리 보고 |
| 조(條) 단위 매칭이 Recall 과대평가(6번 실증) | 조→항/호 정밀 채점(`score_precision.py`) | OLD/NEW 동시 보고 |
| "충돌 서술" judge가 비변별(사전지식으로 둘 다 진술) | 결정론적 flag 기반 P/R/F1 + 과탐율 | 충돌은 flag로 채점 |
| LLM-judge가 사실 진위 미검증 | 수치 근거율·인용 해소가능성(factuality) | 사실성 보조지표 추가 |
| "정답=올바른 추론"인지 미확인 | 골드 경로 대비 경로 충실도 메트릭 | Path 메트릭 신설 |
| 라벨 혼재·무응답 문항 미통제 | 6유형 분류 + answerable 게이팅 | 무응답 분모 분리 |

## 2. 목표(가설)

1. **H1 (핵심)**: 그래프 확장이 **Context Recall**(정답 근거 조항 수집률)을 Naive 대비 높인다.
   - **H1-정밀**: 조 단위뿐 아니라 **항/호 단위 정밀 채점(NEW)** 에서도 우위가 유지된다.
   - **H1-출처**: 우위는 by-construction 의심이 적은 **held_out 문항에서도** 관찰된다.
2. **H2**: hop(추론 단계)이 깊을수록 Naive 누락이 커진다 — *단, 1차에서 반증되어 본 계획에서는
   "hop3 핵심 조문(조특법 §29의8)이 충돌셋 밖이라 앵커 미도달"이라는 구조적 공백 가설로 재정의*.
3. **H3**: CONFLICTS_WITH 엣지가 **충돌 감지(결정론적 flag P/R/F1)** 에 기여한다(서술 judge 아님).
4. **H4**: distractor(동명무충돌·용도상이)에서 **과잉탐지하지 않는다**(과탐율↓).
5. **H5**: 근거 수집 우위가 **Answer Accuracy·Faithfulness**(LLM + 사실성) 우위로 이어진다.
6. **H6 (신설)**: GraphRAG는 정답에 도달할 때 **골드 추론경로(전문가 정의 조문쌍 관계)** 를 실제로
   밟는다 — 즉 **우연일치(coincidence)가 낮고 경로 재현율(path recall)이 높다**.

> **귀인(ablation) 가설**: 위 우위는 "그래프 확장(B2→B3)"과 "개념-충돌 앵커(B3→B4)"라는 **구조적 요인**에서
> 나오며, 프롬프트(B0→B1)·자기수정 루프(B1→B2)만으로는 재현되지 않는다. → §4에서 요인별로 분해.
>
> **[결과로 정밀화됨 — §8 참조]** 실측 결과 이 가설은 *절반만* 맞았다. **우위의 거의 전부는 앵커(B3→B4,
> Recall +0.321)에서 나오고, 앵커 없는 일반 그래프 확장(B2→B3)은 오히려 −0.088로 net 손해**였다. 따라서
> 본 보고서의 결론은 "GraphRAG가 좋다"가 아니라 **"이 법령 도메인에선 *그래프 위에서 작동하는 개념-충돌
> 앵커*라는 구조가 필요하고, 일반 그래프 확장은 보조"** 로 정밀화된다(상세: §8.1, §8.2, §9).

## 3. 실험군 / 대조군 아키텍처 개요

### 3.1 실험군 — GraphRAG 에이전트 (= B4, Ours)
`agent/app.py`의 LangGraph 파이프라인. 5개 노드가 상태(state)를 공유하며 순차 처리하고, Verifier가
REJECT하면 검색으로 회귀한다.

```
질문 → Router → Retriever → Reasoner → Verifier ─(ACCEPT)→ Output
                  ▲                        │
                  └────────(REJECT, ≤2회)──┘
```
- **Router**(gpt-4o-mini): 질문을 simple / multihop / conflict로 분류 → 검색 전략(seed_k·hop·edge_priority) 결정.
- **Retriever**: 벡터 시드 + **개념 구동 충돌 앵커**(질문 개념과 일치하는 CONFLICTS_WITH 정준 조문쌍을
  0.95 최상위로 직접 주입) + 개념 브리지(DEFINES, 문서 간) + REFERENCES 확장 → 재랭킹.
- **Reasoner**(gpt-4o): 수집 컨텍스트만 근거로 답 생성 + 조항 인용 + 충돌 표면화. (충돌 엣지 없으면 flag 강제 false)
- **Verifier**(gpt-4o-mini + 결정론 가드): 인용 존재성(조문 단위 보정)·충돌 미서술을 검사, 실패 시 검색 폭 확대 회귀.
- 임베딩: text-embedding-3-small(Neo4j 벡터 인덱스). *상세 동작 원리는 부록 B.*

### 3.2 대조군 — 강화 Naive RAG (= B0)
`agent/run_eval.py: baseline_context()` + 공정 생성 함수.
- **순수 벡터 검색** + over-fetch 후 같은 조문 중복 제거(다양화)로 한정 예산에 서로 다른 조문 최대 포함.
- **예산 일치**: 청크 수 = GraphRAG 최종 컨텍스트 수 L(최소 8) → 청크 예산 비대칭 제거.
- **생성**: 그래프 Reasoner와 **동일 모델(gpt-4o) 단일 호출**(그래프·앵커·Verifier 미사용).
- 청크·임베딩은 실험군과 **동일** → 변인은 "검색 방식 + 추론 파이프라인"으로 통제.

## 4. Ablation 5모델 상세 설계 (B0~B4)

핵심 아이디어: GraphRAG와 Naive의 격차를 **요인 하나씩** 더해가며 분해해, 차이가 *어디서* 나는지를
수치로 가른다. 구현은 `eval/ablation_ladder.py`이며 **기존 코드 무수정 원칙**(기존 노드/함수를 조합 호출,
앵커 OFF는 `_conflict_by_concept`를 빈 결과로 덮는 비침습 subclass)으로 만들었다.

| 단계 | 컨텍스트(검색) | 생성 파이프라인 | 직전 대비 **유일** 실험변수 |
|------|----------------|------------------|------------------------------|
| **B0** | Naive 벡터(예산 L) | 단순 프롬프트, 단발 | (기준선) |
| **B1** | Naive 벡터(예산 L, 동일) | **Reasoner 프롬프트**, 단발 | 생성 프롬프트 품질 |
| **B2** | Naive 벡터(예산 L, 동일) | Reasoner + **Verifier + 재시도** | 자기수정 루프 |
| **B3** | **그래프 확장(앵커 OFF)** | Reasoner + Verifier + 재시도(B2 동일) | 컨텍스트 출처: 벡터 → 그래프 |
| **B4** | 그래프 확장(앵커 ON)=현행 | Reasoner + Verifier + 재시도(동일) | 개념-충돌 앵커 OFF→ON |

**각 모델의 의미**
- **B0 (순수 벡터 RAG)**: "그래프가 전혀 없을 때"의 최선. 강화 Naive 그대로.
- **B1 (+추론 프롬프트)**: 컨텍스트는 B0와 동일. 답 형식·인용·충돌표면화를 지시하는 Reasoner 프롬프트만 추가
  → **프롬프트 엔지니어링만의 효과**를 분리.
- **B2 (+Verifier 루프)**: 컨텍스트 동일. 인용 존재성·충돌 가드로 REJECT 시 재검색(≤2회)
  → **자기수정 루프만의 효과**를 분리(검색은 여전히 벡터).
- **B3 (그래프 확장, 앵커 OFF)**: 파이프라인은 B2와 완전 동일. 컨텍스트만 **벡터→그래프 확장**으로 교체
  (개념 브리지·REFERENCES 홉·시드충돌은 켜되, **정준 충돌쌍 직접 주입만 끔**) → **그래프 일반 확장의 효과**.
- **B4 (전체, 앵커 ON)**: B3 + **개념 구동 충돌 앵커**만 켠다 → **KG의 핵심 무기(정준 조문쌍 직접 주입)의 순효과**.

**통제변인(전 단계 고정)**: 동일 78문항/gold, 동일 생성모델(gpt-4o)·judge(gpt-4o-mini)·temperature=0,
동일 채점 함수, **동일 컨텍스트 예산 L**(= B4 그래프 컨텍스트 길이) → 예산 비대칭 제거.

**요인 귀인 공식**

```
검색 순수기여   = Recall(B3 또는 B4) − Recall(B0)   # 생성과 무관, 검색만의 효과
프롬프트 기여   = Acc(B1) − Acc(B0)                 # 컨텍스트 동일
재시도 기여     = Acc(B2) − Acc(B1)                 # 컨텍스트 동일
그래프 기여     = 지표(B3) − 지표(B2)               # 파이프라인 동일, 컨텍스트만 그래프
충돌앵커 기여   = 지표(B4) − 지표(B3)               # 앵커만 ON/OFF (KG 핵심 가치)
```

> **B0~B2가 필요한 이유**: "GraphRAG가 좋다"는 주장에 대해 "그건 그래프가 아니라 더 좋은 프롬프트/재시도
> 때문 아니냐"는 반박을 차단한다. B0→B1→B2에서 컨텍스트를 고정한 채 생성 파이프라인만 강화했을 때의
> 변화량과, B2→B3→B4에서 컨텍스트(구조)만 바꿨을 때의 변화량을 비교하면 **구조적 요인의 순기여**가 분리된다.

## 5. 실험 설계

- **평가셋**: `eval/gold_set_final.json` — 사람 G4 전수검수 **78문항**.
  - 유형(6종): 문서간충돌 26 · 단일정밀 18 · 일반 10 · 동명무충돌 10 · 무응답 8 · 용도상이 6.
  - hop: hop1 36 · hop2 34 · hop3 8.
  - 출처: held_out 45 · generated 33. *(상세 분포는 부록 D)*
- **분모 정의(채점 집단)**:
  - 충돌 감지 분모 = **문서간충돌 26**(서로 다른 두 문서 기준 비교).
  - 과탐 분모 = **동명무충돌 10 + 용도상이 6 = 16**(충돌 아님 → flag 켜면 오탐).
  - Recall/Accuracy/Faithfulness 분모 = **answerable=True**(무응답 8 제외 → 공허한 1.0 inflation 차단).
  - 경로 충실도 분모 = **gold_path가 부여돼 required pair가 있는 문항(현재 42)**.
- **절차**: 각 문항을 B0~B4(및 GraphRAG vs Naive)에 동일 입력 → 검색 컨텍스트·생성 답변 수집 → 지표 채점.
- **채점 매칭 단위(이중)**:
  - **OLD(조 단위)**: gold `제26조의8제2항제3호` → `(문서, 제26조의8)`로 정규화 후 청크 `(doc, article)`과 비교.
  - **NEW(항/호 단위)**: `gold_chunk_ids`를 정답으로, 청크가 실제로 그 **항/호 블록**을 포함했는지로 채점(정밀).
  - 지침은 조 구조가 없어 page/section 기준으로 해소.
- **LLM-judge**: Accuracy·충돌 서술 여부는 gpt-4o-mini judge(temperature=0). 충돌 *감지*는 judge가 아니라
  **결정론적 flag**로 채점(서술 judge는 비변별이라 보조로만).
- **사실성(factuality)**: 답변의 수치가 인용 청크/질문에 근거하는지(수치 근거율)와 인용 chunk_id가
  컨텍스트로 해소되는지(인용 해소가능성)를 결정론적으로 검사(`score_factuality.py`).
- **재현성 한계**: 비용·시간 제약으로 **1회 실행**(다회 평균 아님). Router/judge는 소폭 변동 가능 → §9 한계에 명시.

## 6. 평가 메트릭

| # | 지표 | 정의 | 분모 | 채점 방식 | 대응 가설 |
|---|------|------|------|-----------|-----------|
| M1 | **Context Recall (OLD/NEW)** | 컨텍스트가 gold 조항을 포함한 비율(조 / 항·호) | answerable | 결정론 | H1 |
| M2 | **Hop별 Recall / 누락률** | hop 그룹 평균 Recall, 1−Recall | hop 1/2/3 | 결정론 | H2 |
| M3 | **Conflict P/R/F1** | 충돌 flag의 정밀도·재현율·F1 | 문서간충돌 26 | 결정론 flag | H3 |
| M4 | **과탐율(과소응답 포함)** | distractor에서 flag를 켠 비율 / 무응답 미회피율 | 동명무충돌+용도상이 16 | 결정론 flag | H4 |
| M5 | **Answer Accuracy** | 답변이 gold_answer와 핵심 일치(0~1) | answerable | LLM judge | H5 |
| M6 | **Faithfulness** | 답변 주장이 컨텍스트로 뒷받침(0~1) + 사실성(수치근거율·인용해소율) | answerable | judge + 결정론 | H5 |
| M7 | **Path Faithfulness** | path recall(필요 조문쌍 관계 재현) / coincidence(우연일치) / conn precision | required-pair 42 | 결정론(R_gold 대비) | H6 |
| M8 | **요인 귀인 분해** | §4 귀인 공식으로 B0~B4 단계별 Δ | — | 결정론 차분 | 귀인가설 |

> **M7(경로 충실도) 설계 원칙 — 정답으로 가는 길은 하나가 아니다.**
> 같은 정답에 도달하는 유효 경로가 여러 개일 수 있으므로, 노드 시퀀스 완전일치를 요구하지 않는다.
> 대신 ① **조문쌍 관계 단위**로 "전문가가 정의한 필요한 연결(R_gold: CONFLICTS_WITH/DEFINES-bridge/
> REFERENCES)"이 모델 경로에 나타났는지(**path recall**), ② 그 연결이 **정답 도달과 무관하게 우연히**
> 등장했는지(**coincidence**), ③ 모델이 밟은 연결 중 R_gold에 부합하는 비율(**conn precision**)을 분리 측정한다.
> *상세 정의는 부록 C.*

## 8. 결과 분석 — 가설별 판정 (데이터 × 메트릭 × 모델요소)

> 모든 판정은 §7 실측치에 직접 근거한다. "그래프가 다 했다"고 포장하지 않고, **무엇이 효과를 냈고
> 무엇이 효과가 없었는지**를 데이터 성격과 모델요소로 분해해 정직하게 기술한다.

- **H1(근거 수집) — 지지.** Context Recall 0.683 vs 0.436(+0.247). 단, 이 우위는 **개념-충돌 앵커에 거의 전적으로
  귀속**된다(귀인: RecallB4−B3 **+0.321**, RecallB3−B0 **−0.088**). 즉 "GraphRAG가 좋다"가 아니라 **"개념-충돌 앵커가
  좋다"** 로 결론을 좁혀야 정확하다.
- **H1-출처(by-construction 통제) — 부분 지지(잠정).** 앵커 효과는 generated 충돌(+0.491)뿐 아니라 **독립 작성
  held_out 충돌에서도 +0.375**(§7.5-A2) → "정답을 미리 넣어서만"이 아니라 외운 7쌍을 넘어 일반화하는 정황.
  **단 held_out 충돌 n=8로 작아 잠정**(이 수치는 항상 "n=8, 잠정"과 한 문장에서 보고).
- **H2(hop 심화) — 78셋에서 지지 방향.** hop2 +0.38·hop3 +0.29로 깊을수록 격차가 크다. 단 hop3 n=8로 잠정이며,
  hop3 우위는 앵커 의존이 절대적이다(B3 hop3 0.167로 앵커 없으면 급락).
- **H3(충돌 감지) — 지지.** flag&judge B4 0.962, P/R/F1 = 0.641/0.962/0.769. 충돌 감지의 거의 전부가 앵커 기여
  (감지 B4−B3 **+0.616**).
- **H4(과탐 억제) — 실패(가장 큰 약점).** distractor flag 0.875(FP 14). 원인은 **앵커가 유형을 구분하지 않고
  "단시간/상시" 개념만 보면 충돌쌍을 발동**하기 때문(§7.5-A: 충돌 flag 0.96 ↔ distractor flag 0.88, *같은 동전*).
  distractor의 Recall 상승(+0.57)은 **과탐의 부산물**이지 성능이 아니다(§7.5-C, gold가 앵커 주입 조문과 동일).
- **H5(답 품질) — 지지.** Accuracy 0.720 vs 0.447, Faithfulness 0.934 vs 0.743, **Abstention 0.013 vs 0.449**.
  Naive가 근거 부재로 45% "모름"인 데서 격차가 발생(H1의 귀결).
- **H6(경로 충실) — 지지.** conn_precision 0.984·coincidence 0.031. 충돌 문항은 우연일치 0.0으로 *올바른 구조*로 도달.

### 8.1 그래프 도입은 성공이었나 — 정직한 양면 평가
- **성공한 부분**: (1) 문서 간 직접참조가 0인 코퍼스에서 **개념-충돌 앵커가 정답 조문쌍을 확보**해 Recall·충돌감지·
  Accuracy·Abstention을 모두 끌어올렸다. (2) 경로충실도에서 **개념브리지(DEFINES) 엣지가 7회 추론경로에 실제 사용**
  됐고 충돌 문항 우연일치가 0.0 → 우위는 **그래프 구조 위의 탐색**이지 정적 룩업이 아니다.
- **실패/한계**: (1) **앵커 없는 그래프 일반확장(B3)은 net 약손해**(−0.088) — 개념브리지/참조만으론 Naive보다 못하다.
  (2) 앵커는 cross-doc 전용이라 **단일정밀·일반 문항엔 무효이고 일반에서는 과탐만 유발**. (3) 과탐(H4)이 미해결.
- **종합 프레이밍**: 본 결과는 *"GraphRAG 만능"이 아니라 "이 법령 도메인에선 전문가 큐레이션 충돌 엣지를 개념 경유로
  주입하는 구조(앵커)가 핵심이고, 일반 그래프 확장은 보조"* 라는 그림이다. 약점(B3 약손해)을 먼저 인정하고
  **"왜 앵커가 필요했나 = 코퍼스의 참조가 조특에 편중돼 cross-doc 멀티홉이 불가능"**(§7.5-B, 부록 A.3)으로 받으면,
  약점이 곧 설계 정당화가 된다.

### 8.2 "이건 룩업 테이블 아닌가?"에 대한 답 (예상 반론 대비)
- **반론**: "전문가가 정답 충돌쌍 7개를 손으로 만들어 꽂은 룩업 테이블의 승리 아닌가? 그래프(브리지·참조)는
  오히려 방해였는데?"
- **답(데이터 근거)**: (1) 앵커는 고정 답 테이블이 아니라 **질문에서 개념을 감지(MENTIONS/별칭) → CONFLICTS_WITH
  그래프 엣지를 타고 양끝 조문을 가져오는 *탐색*** 이다. 7개 축이지만 **78문항(상당수 held_out)에 일반화**됐고
  held_out 충돌에서도 +0.375. (2) 경로충실도에서 **개념브리지(DEFINES) 7회 사용** → 충돌 엣지 외 그래프 구조도
  추론에 기여. (3) 룩업이라면 우연일치가 높아야 하는데 충돌 coincidence 0.0. → **"구조화된 지식(개념-충돌 엣지)을
  추론 단계에서 주입하는 아키텍처"가 단순 벡터·단순 그래프 모두로는 안 되는 일을 한다**는 것이 본 실험의 기여다.

### 8.3 측정·데이터의 한계 (전수 기록)
1. **과탐(P 0.641, FP 14)** — 명목충돌/실질충돌 미구분. 최대 숙제.
2. **held_out 충돌 n=8** — H1-출처 결론은 잠정. 항상 표본수와 함께 보고.
3. **hop3 n=8** — H2 결론 잠정.
4. **채점 단위** — OLD(조)는 과대평가 가능. NEW(항/호) 병기로 통제(§7.4).
5. **단일 실행** — 다회 평균 아님(Router/judge 확률 변동).
6. **사실성 ≠ 충실성** — judge는 컨텍스트 정합만. 도메인 진위는 원칙적 한계.
7. **충돌 Recall by-construction** — 충돌 문항 gold = 앵커 주입 조문이라 일부 우위는 설계 효과(held_out로 일부 통제).

## 9. 결론

1. **GraphRAG는 강화 Naive 대비 근거 수집(+0.247)·정확도(+0.273)·충실도(+0.191)·무응답(−0.436)에서 우위가 있고,
   올바른 추론 경로로 도달한다(conn 0.984).** (H1·H5·H6 지지)
2. **그 우위의 원천은 "개념-충돌 앵커" 단일 요인이다.** 요인 귀인에서 프롬프트(+0.14)·재시도(+0.02)·앵커없는
   그래프(−0.09)는 작거나 음수이고, 앵커만 Recall +0.321·충돌감지 +0.616을 만든다. 결론은 *"GraphRAG가 좋다"가
   아니라 "이 도메인엔 개념-충돌 앵커라는 구조가 필요하다"* 로 정밀화된다.
3. **앵커는 그래프 위에서만 성립한다(룩업 아님).** CONFLICTS_WITH 엣지 탐색 + 개념 감지로 발동하고, 경로충실도에서
   개념브리지도 함께 쓰였다. 약점인 "앵커 없는 그래프 약손해"는 코퍼스 참조 편중(cross 참조 100% 조특)이 원인으로,
   **앵커 도입의 당위성을 역으로 입증**한다.
4. **미해결 과제는 과탐(H4 실패, FP 14)이다.** 앵커가 유형 무관 발동해 distractor에서 오탐하며, distractor Recall
   상승은 그 부산물일 뿐이다. 명목/실질 충돌 구분이 후속 1순위.

> **한 줄 결론**: *GraphRAG는 "근거를 안 놓치고 올바른 경로로 답한다"를 데이터로 입증했으나, 그 힘은 그래프 일반
> 확장이 아니라 **그래프 구조 위에서 작동하는 개념-충돌 앵커**에서 나오며, 과탐 억제는 남은 숙제다.*

# 부록 B. 에이전트 동작 원리 (코드 기반 논리 풀이)

> 코드를 옮기지 않고, 각 노드가 **무엇을 입력받아 어떤 판단으로 무엇을 내보내는지**를 설명한다.
> (출처: `agent/nodes.py`, `agent/retrieval.py`, `agent/app.py`)

## B.1 전체 흐름과 상태 공유
LangGraph가 단일 `state`(dict)를 5개 노드에 흘려보낸다. 각 노드는 부분 갱신 dict를 반환하고 `trace`에
단계를 누적한다(관찰가능성). 진입점은 Router, Verifier만 분기(조건부 엣지)한다.

## B.2 Router — 질문 → 검색 전략
- gpt-4o-mini가 질문을 **simple / multihop / conflict** 중 하나로 분류(JSON 강제).
- 분류에 따라 검색 파라미터를 결정:
  - `simple`: seed_top_k 4, max_hops 1, **edge_priority=[concept]**, ctx 6.
  - `multihop`: seed 5, hop 2, edge_priority=[references, concept], ctx 12.
  - `conflict`: seed 5, hop 2, **edge_priority=[conflict, concept, references]**, ctx 12.
- **설계 의도**: `simple`에는 conflict/references가 없으므로 **충돌 앵커가 발동하지 않는다** →
  단순 질의에서 충돌을 끌어와 과잉탐지하는 것을 라우팅 단계에서 차단.

## B.3 Retriever — 하이브리드 검색(GraphRAG 핵심)
입력 질문으로 다음을 **점수 사전(scored)** 에 합치고 상위 max_ctx개를 컨텍스트로 확정한다.
1. **벡터 시드**: 임베딩 → Neo4j 벡터 인덱스 top-k. (각 시드 score = 코사인 유사도, via=seed)
2. **개념 구동 충돌 앵커(0순위, 최강)**: 질문에서 개념(상시근로자·단시간·피보험자 등, 구어 별칭 보강)을
   감지하고, 그 개념과 일치하는 **CONFLICTS_WITH 정준 조문쌍의 양끝을 score 0.95로 직접 주입**.
   - **단, `edge_priority`에 conflict 또는 references가 있을 때만 발동**(simple 제외) → 과탐 방지.
   - 의의: 벡터가 부수 조문만 잡아도, **KG가 정답 충돌쌍을 구조적으로 보장**.
3. **CONFLICTS_WITH(시드 직접 연결)**: 시드와 연결된 충돌 상대 청크를 0.5+0.3로 부스트.
4. **개념 브리지(문서 간)**: 시드가 가리키는 Concept을, **상대편 문서의 DEFINES(권위 정의) 청크**로 연결
   (score 0.58). 문서 간 다리는 이 경로뿐(부록 A.3).
5. **REFERENCES(인용 사슬)**: 시드가 인용한 Article의 청크를 확장(0.55).
- **재랭킹**: score 내림차순 상위 max_ctx개 확정. (앵커 0.95가 항상 상단)
- **경로 트리플 복원(관찰가능성, 로드맵 5)**: 컨텍스트 확정 *후* 시드→엣지→컨텍스트를 node→edge→node로
  사후 복원한다. **점수·컨텍스트에 영향 없음**(LIMIT 없는 사후 조회) → 경로 메트릭(M7)의 입력.

## B.4 Reasoner — 컨텍스트 기반 답 생성
- gpt-4o가 **수집 컨텍스트만** 근거로 답·인용(chunk_id)·충돌서술을 JSON으로 생성.
- **결정론적 게이팅**: 검색에서 CONFLICTS_WITH 엣지가 하나도 없으면 `conflict_flagged`를 **강제 false**.
  → 모델이 컨텍스트 없이 충돌을 단정하는 환각을 차단(과탐 억제의 1차 방어선).

## B.5 Verifier — 검증 + 자기수정 회귀
- 먼저 **인용 보정(#10 함정 완화)**: 인용 chunk_id가 컨텍스트에 정확히 없어도, **조문 단위(`근기령_제7조의2`)
  로는 일치**하면 실제 컨텍스트 chunk_id로 치환(항 표기 차이를 흡수).
- gpt-4o-mini가 근거 존재성·수치 충실성을 1차 판정한 뒤, **결정론적 가드 2종으로 덮어쓴다**:
  - 가드1: 조문 단위로도 컨텍스트에 없는 인용이 남으면 → **REJECT**(환각 인용).
  - 가드2: 충돌 엣지가 있는데 답이 충돌/차이를 안 다루면 → **REJECT**(누락).
- REJECT면 `retry_count`+1 후 **검색으로 회귀**. 단 **회귀 시 탐색 '폭(k)'만 키우고 route 의미(edge_priority)는
  바꾸지 않는다** → simple distractor가 재시도로 conflict로 변질돼 과탐하는 것을 방지. 최대 2회.

## B.6 "이 구조 때문에 Naive와 갈린다"의 요지
- **수집(Recall)**: 벡터가 놓치는 정답 조문을 **개념 앵커·개념 브리지가 구조적으로 끌어온다**(B3→B4 핵심).
- **신뢰성(과탐/환각)**: simple 라우팅·flag 강제 false·인용 가드가 **distractor 과탐과 환각 인용을 차단**.
- **경로(M7)**: 답이 맞을 뿐 아니라 **전문가 정의 관계(CONFLICTS_WITH/DEFINES-bridge)** 를 실제로 밟는다.

# 부록 C. 도입 메트릭 상세 (왜·어떻게)

## C.1 Context Recall — OLD(조) vs NEW(항/호)
- **OLD**: gold 조항을 `(문서, 조)`로 정규화 → 컨텍스트의 `(doc, article)` 집합에 포함되면 hit.
  - 한계: 같은 조 안 아무 청크나 들어와도 hit → **과대평가**(1차 6번이 실증: doc-level hit인데 오답).
- **NEW**: `gold_chunk_ids`(정답 항/호 블록)를 기준으로, 컨텍스트가 **그 블록을 실제로 포함**했는지로 채점.
- **둘을 병기**해 "조는 맞췄지만 정작 필요한 항/호는 놓친" 경우를 드러낸다. (정밀화 = 로드맵 1)
- **두 채점기 일원화(질문 대비)**: `score_precision`과 `run_experiment`의 OLD가 0.338 vs 0.683으로 어긋났던
  원인은 `score_precision.PREFIX_DOC`가 `조특령→"조세특례제한법시행령"`(정식 풀네임)으로 매핑한 반면 그래프
  실제 `Chunk.doc`는 `"조특법시행령"`이라 **법령 doc 매칭이 전부 실패**한 버그였다(고보·근기·지침은 정상).
  수정 후 OLD 0.721로 `run_experiment`(0.683)과 수렴(§7.4). → 우위 방향은 동일했고, *절대값 정합성*만 복원됨.

## C.2 Conflict P/R/F1 + 과탐율 (로드맵 4)
- 충돌 *감지*는 **결정론적 flag**로 채점한다(judge 서술은 사전지식으로 비변별 → 보조).
  - Precision = (옳게 켠 flag)/(켠 flag 전체), Recall = (옳게 켠 flag)/(실제 충돌 26), F1 = 조화평균.
- **과탐율**: distractor(동명무충돌+용도상이 16)에서 flag를 켠 비율. 명목충돌(이름만 같은 지표)과
  실질충돌을 구분 못 하는 구조적 약점을 직접 수치화.
- **과소응답률(via 귀인)**: 충돌 근거를 어떤 엣지(앵커/시드충돌/브리지/참조)로 가져왔는지 귀인하고,
  근거 부재로 답을 못 낸 비율을 분리.

## C.3 Faithfulness + 사실성 (로드맵 2)
- judge Faithfulness(0~1) = 답 주장이 컨텍스트로 뒷받침되는가.
- **사실성 보강(결정론)**: ① 수치 근거율 = 답의 수치가 인용 청크/질문에 실재하는 비율,
  ② 인용 해소가능성 = 인용 chunk_id가 컨텍스트로 해소되는 비율. → judge가 못 보는 "근거 없는 수치"를 포착.

## C.4 Path Faithfulness — 비유일 경로를 감안한 설계 (로드맵 7)
**문제**: 같은 정답에 이르는 유효 추론 경로가 여러 개라, "골드 경로와 노드 시퀀스 완전일치"를 요구하면 부당.
**해법**: 조문쌍 **관계(R)** 단위로 비교한다.
- `R_gold` = 그 문항의 정답 도달에 **필요한 조문쌍 관계 집합**(전문가 gold_path → CONFLICTS_WITH /
  개념브리지(DEFINES) / REFERENCES 관계로 환산). gold_path의 양끝이 모두 그 문항 gold 조문일 때만 채택
  (phantom pair 방지).
- 모델이 밟은 경로(B.3의 트리플)에서:
  - **path recall** = R_gold 중 모델 경로에 등장한 비율(필요한 연결을 실제로 밟았나).
  - **coincidence(우연일치)** = 정답 도달과 무관하게(노드 미완전도달인데) 관계만 등장한 비율 → 낮을수록 좋음.
  - **conn precision** = 모델이 밟은 관계 중 R_gold에 부합하는 비율(헛디딤 적을수록 높음).
- **최종셋 결과**: path_recall 0.698 · coincidence 0.031 · conn_precision 0.984 → 우연이 아니라 **구조적으로** 도달.

# 부록 F. 슬라이드별 세부 연출 지시서

> **이 부록을 쓰는 이유**: 메트릭이 7개라 발표가 "지표 나열 → 그래서 뭔데?"로 끝나기 쉽다.
> 아래 모든 슬라이드는 **단 하나의 서사 골격**을 따른다. 이 골격을 벗어나는 자료/멘트는 넣지 않는다.

## F.0 관통 서사(전 슬라이드 공통 규칙)

**서사 골격 — "가설 → 그래서 이렇게 만들었다 → 그래서 이렇게 검증한다 → 맞았다/틀렸다"**
1. 우리는 *이런 가설*을 세웠다 (신뢰할 수 있는 답 = 근거 안 놓침 + 올바른 경로).
2. 그 가설이 맞으려면 시스템이 *이렇게 생겨야* 한다 → 그래서 **그래프·개념-충돌 앵커**를 만들었다.
3. 그 가설을 *공정하게 증명*하려면 → 그래서 **B0~B4 사다리 + 78문항 3축 + 7메트릭**을 준비했다.
4. 실험해보니 가설이 *맞았다/틀렸다* → 맞은 건 메커니즘으로, 틀린 건 한계로 정직하게.

**메트릭-가설 1:1 매핑(절대 섞지 말 것)** — 발표에서 메트릭은 *절대 단독으로* 말하지 않는다.
항상 "이 가설을 보려고 이 메트릭을 본다"로 묶는다.

| 가설 | 한 줄 주장 | 그것만 보는 메트릭 | 슬라이드 |
|------|-----------|---------------------|---------|
| **H1** 근거 수집 | "정답 조항을 안 놓친다" | **M1 Context Recall(OLD/NEW)** | S6 |
| **H2** hop 심화 | "복잡할수록 격차가 커진다" | **M2 hop별 Recall/누락률** | S6 |
| **H5** 답 품질 | "근거 우위 → 답 품질 우위" | **M5 Accuracy · M6 Faithfulness · Abstention** | S6 |
| **귀인** 원인 | "우위는 (그래프 아님) **앵커** 때문" | **M8 요인 귀인 분해(waterfall)** | S6 |
| **H3** 충돌 감지 | "충돌을 잡아낸다" | **M3 Conflict P/R/F1** | S7 |

> **꼬임 방지 3원칙**:
> ① M3와 M4는 **하나의 동전**이다(같은 flag의 Recall vs Precision) → S7에서 "재현율은 높지만 정밀도가 약점"
>    한 문장으로 묶는다. 따로 떼어 두 번 설명하지 않는다.
> ② M5·M6·Abstention은 H1의 **귀결**이다 → "근거를 모았으니(H1) 답이 좋아졌다(H5)"는 *인과 한 줄*로 연결.

> ④ **"그래프 vs 앵커"를 절대 뭉뚱그리지 말 것** → 우위 원천은 앵커(B3→B4)이고 앵커없는 그래프(B3)는 손해.

  충돌 우연일치 0% → 구조적 도달(H6). 충족 엣지에 개념브리지가 7번 → 룩업이 아니라 **그래프를 타는 추론**."

## S9 — 핵심 통찰 : 앵커는 "그래프 위 구조"이고, 그게 이 도메인에 필요했다 (1:30)

> **발표의 클라이맥스.** 양면적 결과(앵커=우위, 일반그래프=손해)를 *약점이 아니라 설계 정당화*로 뒤집는다.

- **자료 — 한 장 정리(텍스트 3블록 + 작은 도식).**
  - 블록1 "우위의 원천": 워터폴 재인용 — 앵커 +0.32 / 일반그래프 −0.09.
  - 블록2 "왜 일반그래프는 약했나": **cross 참조 100% 조특**(부록 A.3) → 문서 간 멀티홉이 참조로 불가 →
    hop3 충돌에서 B3=0(§7.5-B). *그래서* 개념 경유 충돌 주입(앵커)이 필요.
  - 블록3 "룩업이 아니다": 7쌍이 78문항(held_out 포함)에 일반화 + 개념브리지 7회 사용 + 우연일치 0.0.
- **발표 흐름(90초)**: "정직하게, *일반 그래프 확장은 이 코퍼스에서 손해*였습니다 — 참조가 조특에만 몰려
  문서 간을 못 잇기 때문. **바로 그래서** 개념 경유로 충돌쌍을 주입하는 앵커를 설계했고 그게 우위의 전부입니다.
  이건 정답 7개를 외운 룩업이 아니라, 7쌍이 78문항에 일반화되고 개념브리지까지 추론에 쓰인 **구조화된 지식의 탐색**입니다."
- **반드시 이해(룩업 반론 풀버전)**: E.4 첫 Q&A 답변을 그대로.

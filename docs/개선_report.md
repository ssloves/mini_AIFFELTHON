# 로드맵 1번 실행 보고 — 채점 정밀화 (조 → 항/호 + 지침 page/section)

> 대상: 평가 메트릭(2순위) 중 "Context Recall 채점 정밀화 + 6번 가짜 hit 제거".
> 원칙 준수: **모델/그래프(agent, graph)·데이터 라벨은 수정하지 않음.** 측정 도구만 신설.
> 신설 파일: `eval/score_precision.py` · 산출물: `eval/_precision_results.json`.

## 1. 주요 변경점

1. **매칭 단위 강화**
   - 법령: `(문서, 조)` → **`(문서, 조, 항범위 교집합)`**. gold의 "제N항"과 청크 chunk_id 접미사
     (예: `조특령_제26조의8_제1-2항`)의 항 범위를 파싱해 교집합으로 판정.
   - 지침: 문서단위 와일드카드(`*`) → **정밀 앵커(page/section) 집합 매칭**.
2. **6번 정밀 정답 라벨 확정**: "정부·지자체 인건비 지원액 차감" 규정의 실제 위치를 원문에서 찾아
   **`지침_p43_s96`(3-4 중복지원 제한 본문) / `지침_p143_s312`(서식 ⑧ 체크리스트)** 로 라벨링.
3. **시스템 RETURN 변경 없음**: 항/페이지/섹션은 전부 `chunk_id`에서 파싱 → 검색·점수·재랭킹 불변.
4. (환경 복구) Neo4j 컨테이너가 내려가 있어 Docker Desktop + `graph/docker-compose.yml`로 재기동 후 측정.

## 2. 논리 (왜 이렇게 하나)

- 기존 채점은 "같은 **조**의 아무 청크나 = hit", "지침 아무 청크나 = hit"라 **틀린 청크인데 정답으로 집계**
  되는 가짜 hit가 있었다(exp_report 6번: Recall 1.0인데 Accuracy 0).
- 항/페이지 단위로 좁히면 "정작 필요한 조항·규정이 컨텍스트에 있었는가"를 보게 되어, Recall이 **답변
  정확도와 다시 정렬**된다.
- 단, **청크가 항 단위로 분할된 경우에만 분해능**이 생긴다(조 단위 청크·부칙 동명조는 와일드카드로
  인정). 이 한계는 §4에 명시한다.

## 4. 부수 발견 (정직 고지)

- **5번 — gold 과다라벨**: 제2항(제외대상)이면 답이 성립하는데 gold가 제9·13항(원천징수 확인 절차·
  특수관계인 정의)까지 포함. 정밀화가 *모델 실패가 아니라 gold 라벨 과다*를 드러냄 → **gold 큐레이션
  필요**(데이터 축 과제). G_new 0.33의 일부는 모델이 아니라 라벨 탓.
- **4·8번 — 그래프 우위의 정직한 강화**: 그래프는 필요한 항-세그먼트(제2항·제7항 등)를 모두 확보,
  Naive는 같은 조의 *다른 항-청크*만 잡아 0.5로 하락. 정밀화가 둘을 제대로 구분.
- **11번 — 오라우팅 손실 노출**: simple로 잘못 분류된 11번이 근기 제7조의2 중 **핵심인 제4항(제3-4항
  청크)을 못 가져옴**을 정밀화가 드러냄(0.67 → 0.33). OLD는 이를 숨기고 있었음.
- **잔여 한계**: ① 항 분해능은 *항 분할 청크에서만* 작동(조 단위·부칙 동명조는 와일드카드 인정).
  ② 지침 정밀 라벨은 **6번만 완료**, 나머지 5문항(3·7·8·10·18)은 아직 loose(`*`) → 과대평가 잔존.
  ③ 본 측정은 Context Recall 전용(생성/judge 미호출) → Accuracy·Faithfulness는 영향 없음.

# 로드맵 2번 — judge 사실성 체크 + '17번 오염' 가설 검증

> 산출물: `eval/score_factuality.py`, `eval/_factuality_results.json`.
> 원칙: **측정 도구만 신설. 모델/그래프 무수정.** 저장된 답변(`eval/_exp_results.json`의 `g_ans`/`n_ans`)을
> 재채점해 비용·재현성을 확보(에이전트 재실행 불필요).

## 6. 주요 변경점

기존 Faithfulness는 LLM judge 단일 점수(컨텍스트-답변 일관성)였다. 이를 **근거 귀속(grounding) 기반의
3중 사실성 메트릭**으로 분해·정밀화했다.

| 항목 | 기존 | 신설(로드맵 2) |
|---|---|---|
| (A) 수치 근거율 | 없음 | **결정론**: 답변의 의미있는 수치(세/시간/년/명/원/%)가 인용 청크 text에 실존하는 비율 |
| (B) 엄격 LLM 지지도 | 컨텍스트 전체 기반 | **인용 청크 text에만** 근거해 핵심 주장 지지(모수지식 사용 금지) |
| (C) 인용 해소성 | 없음 | 답변의 `[chunk_id]`가 KG에 존재(조단위 폴백)하는지, **미해소 인용=환각 의심** 분류 |

## 7. 논리 — 왜 이렇게 바꿨나

- **"사실성"의 조작적 정의**: 외부 진리표 없이도 "답변의 수치/주장이 *실제 검색·인용된 근거*에 실재하는가"는
  결정론적으로 검증 가능하다. 환각은 곧 "근거에 없는 수치/주장"이다. LLM judge보다 재현성이 높고 변호 가능하다.
- **'17번 오염'은 가설일 뿐 — 증거로 판정**: exp_report는 17번 답변의 "군필 39세 / 조특령 병역 차감"을
  *verified=false 세트3 오염*으로 단정했다. 하지만 이는 **청크 대조 이전의 낡은 가정**이었다. 메트릭으로
  인용 청크 원문을 직접 대조해 참/거짓을 가린다.

## 8. 결과

### 8.1 '17번 오염' 가설 → **반증(거짓)**

17번 graph 답변이 인용한 청크를 원문 대조한 결과:

- 인용 = `지침_p29_s64`, `조특령_제26조의8_제1-2항`, `조특령_제26조의8_제3-4항` → **3개 모두 KG에 실존**.
- `지침_p29_s64` 원문: *"…만 15~34세… 군필자의 경우 의무복무기간에 비례하여… (최고 만 39세로 한정)"* →
  **"군필 39세"는 인용 청크에 실재**. 미근거 수치 0건, 수치근거율 1.0.
- 즉 **17번은 오염이 아니다.** Faithfulness judge가 1.0을 준 것은 (적어도 근거 충실도 면에서) **정당**했다.
- 17번의 *진짜* 약점은 사실성이 아니라 **distractor에서의 관련성/과탐**(질문은 계약 2년·주40h → 포함 여부인데
  연령 기준까지 끌어와 충돌 프레이밍)이며, 이는 **사실성이 아니라 과탐율(Conflict Over-detection) 메트릭 영역**이다.
- → **조치**: 차단 블록리스트는 *불필요/부적절*(존재하지 않는 오염을 인위로 깎게 됨). 대신 `docs/exp_report.md`의
  17번 실패사례 기술(“verified=false 오염”)을 **사실 정정** 대상으로 표시(아래 9절 승인 요청).

### 8.2 사실성 메트릭 집계(18문항, 아티팩트 보정 후)

| 지표 | GraphRAG | Naive | 비고 |
|---|---|---|---|
| (A) 수치 근거율 | **1.000** | 1.000 | 두 군 모두 근거 없는 수치 환각 없음 |
| (B) 엄격 LLM 지지도 | **1.000** | 0.867 | 인용 청크에만 근거. Naive가 17·6·13에서 하락 |
| (C) 미해소 인용 | **0건** | — | graph 인용은 조단위 정규화 시 전부 KG로 해소 |
| (참고) 기존 faith | 0.933 | 0.717 | LLM 단일 judge — 잡음 큼 |

- **그래프 답변에는 환각/오염이 검출되지 않았다**(A·B·C 모두 통과). 기존 LLM faith 0.933은 judge 잡음이었고,
  결정론 기준으로는 그래프 답변이 근거에 *완전히* 충실하다.
- **Naive 엄격지지도 0.867 < Graph 1.000**: 특히 17번에서 Naive는 근거 없는 답(지지도 0.0)을 생성 →
  사실성 축에서도 그래프가 우위.

### 8.3 부수 발견 — 사실성 채점기의 *위양성*(메트릭 고도화의 실질 산출물)

1차 실행 시 1·13·14·18번이 "환각 의심"으로 오탐되었고, 원인을 추적해 보정했다. 이 보정 자체가 평가 인프라의
고도화다:

- **인용ID 형식차(위양성)**: 1번은 답변이 `[고보령_제3조_제1항]`으로 인용했으나 실제 chunk_id는 `…_제1-2항`
  (항 범위 청크). 정확 일치 검사는 이를 *환각*으로 오탐 → **조단위 폴백(`art_key`)** 도입(프로젝트의 Verifier
  조단위 정규화와 동일 규칙). 보정 후 미해소 0건.
- **질문 인용 수치(위양성)**: 13·14·18의 `50시간`·`14시간`·`4.8명`은 **질문문에 그대로 등장하는 전제 수치**.
  답변이 질문을 복창한 것을 환각으로 오탐 → **질문 텍스트를 근거에 포함**하도록 보정.
- **연도 토큰(잡음)**: `2024년` 같은 연도가 수량 수치로 잡힘 → 연도 스트립 보강.
- 교훈: **결정론 사실성 채점은 (a) 인용 조단위 정규화, (b) 질문 수치 화이트리스트가 없으면 위양성을 낸다.**
  이는 평가메트릭 정의서에 채점 전제로 명문화해야 한다.

## 12. (F) 로드맵 3번 — ablation 사다리 B0~B4

> 산출물: `eval/ablation_ladder.py`, `eval/_ablation_results.json`.
> **모델/그래프 무수정.** 앵커 OFF(B3)는 `retrieval.py`를 고치지 않고 `_conflict_by_concept`를 빈
> 결과로 덮는 **비침습 subclass(`NoAnchorRetriever`)** 로 구현. 나머지 부품은 기존 노드 그대로 조합.

### 12.1 설계 — 통제변인(고정)과 실험변수(유일 차이)

핵심 원칙: **인접 단계 사이에는 단 하나의 변수만 바뀐다.** 컨텍스트 예산은 전 단계 **L(=B4 그래프 컨텍스트
길이)로 고정**해 "청크 수 비대칭"이라는 교란을 원천 제거했다.

| 전이 | 통제변인(공정하게 고정) | 핵심 실험변수(유일 차이) |
|------|------------------------|--------------------------|
| **B0→B1** | 컨텍스트(Naive 벡터, 예산 L 동일)·생성모델(gpt-4o)·단발·judge | 생성 프롬프트(단순 → Reasoner: 인용·충돌서술 지시) |
| **B1→B2** | 컨텍스트(동일)·생성 프롬프트(Reasoner)·모델 | Verifier + 재시도 루프 유무 |
| **B2→B3** | 생성 파이프라인(Reasoner+Verifier+루프) 전부 동일 | 컨텍스트 출처: Naive 벡터 → **그래프 확장(앵커 OFF)** |
| **B3→B4** | 그래프 검색 파이프라인·route·params·재시도 폭 전부 동일 | **개념-충돌 앵커 OFF→ON** |

- 전 단계 공통 고정: 동일 18문항/gold, 동일 채점 함수(조 단위 recall), temperature=0.
- B3·B4는 **같은 router 결과(route/params)** 를 공유 → 둘의 차이는 *오직 앵커 주입 여부*.
- 검증된 통제 무결성: 결과에서 **B0=B1=B2의 Recall이 0.574로 정확히 동일** → "컨텍스트 동일" 통제가
  실제로 작동했음을 데이터가 증명(생성만 바뀌면 검색 지표는 불변).

### 12.3 해석 — *데이터에 근거한* 결론 (논리 비약 금지)

1. **그래프의 Recall 우위는 전적으로 '개념-충돌 앵커' 단일 요인에서 나온다.** 앵커를 끈 그래프 확장(B3)은
   오히려 강화 Naive보다 Recall이 **낮다(0.417 < 0.574)**. 브리지·참조 확장이 한정 예산(L)에서 다양화된
   벡터 시드를 밀어내기 때문이다. 앵커가 켜지면(B4) 0.815로 급등(+0.398). → "그래프라서 잘 찾는다"가 아니라
   **"정준 충돌쌍을 직접 주입하는 앵커라서 잘 찾는다."**
2. **Answer Accuracy 격차(B0→B4 +0.373)의 최대 기여자는 그래프가 아니라 *생성 프롬프트*(+0.217)다.**
   재시도 +0.083, 그래프확장 +0.023, 앵커 +0.05. 즉 그래프·앵커의 *순수 Accuracy* 기여 합계는 +0.073에
   불과하다. → **"Accuracy +0.37이 그래프의 구조적 이점"이라는 서술은 과대해석.** 그래프의 가치는
   Accuracy가 아니라 **Recall(+0.398)과 충돌 감지(+0.714)** 에 있다.
3. **충돌 감지(결정론 flag)는 앵커의 고유 기제다(+0.714).** judge-only 충돌 서술은 B0조차 0.857로 비변별
   (생성모델 사전지식) — 즉 "충돌을 *말하는가*"가 아니라 "정준 두 조문을 *실제로 수집·인용해 결정론적으로
   플래그하는가*"가 시스템을 가른다.
4. **앵커는 양날의 검(정직한 트레이드오프).** 앵커는 충돌 감지(+0.714)를 주지만 distractor 과탐도 함께
   유발(B3=0.0 → B4=0.5). B3은 과탐 0이나 감지도 0.14. → 앵커의 순효과는 "감지 이득 vs 과탐 비용"의 교환.

### 12.4 사용자 질문에 대한 직답 — "차이가 정말 추론방식의 차이뿐인가?"

**아니다(데이터 기준).** Naive의 낮은 Accuracy(0.433)는 *절반 이상이 약한 단순 프롬프트* 탓이다 —
같은 Naive 컨텍스트에 Reasoner 프롬프트만 입혀도(B1) 0.65로 오른다. 따라서 "공정한" 구조 비교는
**B1~B4 사이**에서 봐야 하며, 그 구간에서 그래프(검색 방식)의 *순수* Accuracy 기여는 작다(+0.073).
그래프의 진짜 구조적 이점은 **근거 수집률과 결정론적 충돌 감지**에 국한되며, 그 기제는 *개념-충돌 앵커*
단일 부품에 집중돼 있다. (이는 exp_report의 "구조적 우위" 주장을 *반박이 아니라 정밀화* 한다 — 우위의
**위치를 Accuracy에서 Recall·충돌로 옮긴다**.)

## 14. via 귀인 — "그래프가 없었으면 놓쳤을 근거"의 정량화

컨텍스트 청크의 `via` 라벨(seed=벡터 / 그 외=그래프 경로)로, 각 gold 조문이 **벡터 시드로 잡혔는지,
그래프 전용 경로로만 도달했는지**를 분해(전체 29개 gold 토큰).

| 분류 | 토큰 수 | 비율 | 의미 |
|------|--------|------|------|
| seed_only | 4 | 13.8% | 벡터만으로 도달(그래프 불필요) |
| both(seed+graph) | 4 | 13.8% | 둘 다 도달 |
| **graph_only** | **15** | **51.7%** | **벡터로는 못 잡고 그래프 전용 경로로만 도달 → 그래프 한계기여** |
| missed | 6 | 20.7% | 어느 경로로도 미도달(= Recall 천장) |

- **gold 근거의 절반 이상(51.7%)이 그래프 전용 경로(주로 개념-충돌 앵커)로만 도달했다.** 벡터 시드만으로
  잡힌 건 8/29(27.6%)뿐. → 로드맵 3번의 결론("Recall 우위는 앵커 단일 요인")을 *문항 근거 수준에서* 재확인.
- **missed 6건이 Recall 천장**: 11번(route=simple로 앵커 미발동), 3·7번(hop3 연쇄추징 핵심 조문이 충돌셋
  밖 — by-construction 일반화 갭), 14번(어휘 간극), 16·17번(distractor). → 데이터 확장(로드맵 6) 대상이
  *문항별로* 특정됨.
- ablation B3<B0(앵커 끈 그래프 확장이 오히려 손해)과 정합: 이득은 *일반 확장*이 아니라 *앵커가 끌어온
  graph_only 15건*에서 나온다.

## 15. 충돌 혼동행렬 (P/R/F1) — "오탐률 0.5"보다 풍부한 신뢰성 분해

cross-doc 충돌 7(양성) vs distractor 6(음성)에서 결정론 `conflict_flagged`의 혼동행렬:

| | 예측=충돌 | 예측=무충돌 |
|---|---|---|
| **실제 충돌(7)** | TP=6 | FN=1 |
| **실제 무충돌(6)** | FP=3 | TN=3 |

- **Precision 0.667 · Recall 0.857 · F1 0.75.**
- **FN 1건 = 11번**(cross-doc인데 route=simple로 앵커 미발동 → 미플래그). 라우팅 손실이 충돌 *재현율*을
  직접 깎음을 수치로 확인.
- **FP 3건 = 13·16·17번 distractor 과탐**. 충돌 *정밀도*를 깎는 원인이 distractor 오탐임을 분리.
- 단일 "오탐률 0.5"가 사실은 **재현율 0.857(라우팅이 깎음) + 정밀도 0.667(과탐이 깎음)** 의 두 축으로
  분해됨 → 개선 타깃이 (a) 라우팅(11번)과 (b) 명목충돌 게이팅(13·16·17)으로 명확히 갈린다.

## 16. 과소응답률(abstention) — Recall↔회피의 교환관계

답변이 "모름/근거 없음"으로 회피한 비율:

| | 전체 | conflict 문항 |
|---|---|---|
| GraphRAG | **0.0** | 0.0 |
| Naive | **0.5** | 0.5 |

- **Naive는 충돌 문항의 절반에서 "모릅니다"로 회피**한다(앵커가 없어 근거를 못 찾음). GraphRAG는 회피 0.
- 중요한 정직성 포인트: **Naive의 낮은 Accuracy는 상당 부분 *환각*이 아니라 *정직한 회피* 다.** 즉 Naive는
  "약한 대조군(아무 말이나 지어내는)"이 아니라 "근거 없으면 모른다고 하는 견고한 대조군"이며, 격차는
  "Naive가 틀린다"가 아니라 **"Naive는 근거를 못 찾아 답을 *못* 한다"** 에서 발생한다(로드맵 3 결론 보강).

## 17. 설계 — 왜 점수 불변(관찰가능성)인가

사용자 요청("어떤 노드에서 어떤 엣지를 타고 어떤 노드로 갔는지 로그")을 직접 구현. **동결 원칙과 무관하게
진행 가능한 *관찰가능성* 변경**임을 보장하는 방식:

- `_path_triples`는 **컨텍스트(`ranked`)가 *확정된 뒤*** 시드∪컨텍스트 ↔ 컨텍스트 사이의 관계를 *사후 조회*해
  복원한다. `context`/`conflicts`/`score` 계산 코드를 일절 건드리지 않으며 LIMIT도 없다 → **검색 결과·재랭킹·
  최종 답에 영향 0**(설계상 불변). 데모 실행에서 기존과 동일한 route·컨텍스트·충돌쌍이 재현됨을 확인.

## 18. 결과 — 복원된 경로(예시)

세 가지 엣지 유형 모두 node→edge→node로 복원된다(`eval/_path_log.json`).

- **CONFLICTS_WITH(직접, axis/concept 포함)** — 예: 1번

## 20. 설계 원칙 — "생성으로 무결"이 아니라 "게이트 통과로 무결에 수렴"

가이드 §0을 그대로 코드화했다. 무결성을 **세 개의 다른 축**으로 분리해 각각 막는다.

| 축 | 막는 게이트 | 구현 방식 |
|----|------------|-----------|
| 라벨 무결성(gold가 맞나) | G1·G2·G3·G5 | provenance 대조 / 수치 grounding / 도출판정 / 중복제거 |
| 엣지 무결성(KG 엣지가 맞나) | **G0** | 기존 `reference_audit` 재사용(REFERENCES 90.8%) |
| 순환 무결성(by-construction 아닌가) | **G6** | `origin∈{generated,held_out}` 강제 + 분리집계 |

게이트는 **두 종류로 분리**했다(정직성): 결정론 게이트(G0·G1·G2·G5·G6, 코드가 pass/fail 확정)와
판단 게이트(G3·G4, LLM/사람 — 자동은 *proxy* 표기). G4(2인 교차검수)는 자동화 불가라 **proxy 미수행·사람 몫**으로 명시.

## 21. 핵심 부품 — chunk resolver (게이트 논리의 린치핀)

held-out·시드의 조항 표기는 KG 실제 chunk_id와 **입자도가 다르다**. KG 청크는 **항 범위**로 묶여 있다:
`조특령_제26조의8_제{1-2 / 3-4 / 5-8}항`, `고보령_제3조_제1-3항`. 반면 사람은 `제2항`·`제4항`으로 인용한다.

→ resolver가 (문서접두사, 조, **항 범위 포함관계**)로 정규화한다. 예:
- `조특법시행령 제26조의8제2항제2호` → `조특령_제26조의8_제1-2항` (2항이 1-2 범위에 포함)
- `고용보험법시행령 제3조제1항` → `고보령_제3조_제1-3항` (**부칙의 동명 `고보령_제3조` 배제** — body 우선)
- 지침은 조가 없어 **원문 대조 완료분만** 키워드→청크 화이트리스트(`지침_p36_s77`/`p25_s52`/`p43_s96`).

**이 resolver가 없으면 항 표기 차이가 전부 G1 위양성(가짜 폐기)이 된다.** 즉 게이트의 "엄격함"은
resolver의 "정규화 정확성"과 짝을 이뤄야 무결성과 수량을 동시에 지킨다.

## 28. 설계 — "같은 답에 이르는 길은 하나가 아니다"를 어떻게 단단히 다뤘나

문제의식(사용자 지적): CONFLICTS_WITH 직접 / 개념 브리지 / 참조 사슬은 **같은 두 노드를 잇는 동치
경로**다. 모델 traverse를 단일 gold 경로와 *정확히* 비교하면 유효한 대안 길을 부당하게 깎는다.

해법 — **3계층 분리**:

| 계층 | 무엇 | 출처(중요) |
|------|------|-----------|
| 노드 | gold 조문(법적 근거) | Context Recall이 측정 |
| **관계** | R_gold = gold 노드 사이 *필수 연결* | **전문가 근거**(큐레이션 충돌쌍 + 작성 gold_path) |
| **경로** | 모델이 실제 traverse한 길 | 로드맵5 경로 로깅(조문쌍으로 축약) |

**핵심 두 장치:**
1. **경로 비유일성 흡수** — 판정은 *조문쌍* 단위. 같은 쌍을 **어떤 엣지로 잇든**(CONFLICTS_WITH ≡
   개념브리지 ≡ 참조) 동일 관계로 인정. → 동치 경로를 깎지 않는다.
2. **순환(by-construction) 회피** — *요구관계*(R_gold)는 **전문가 큐레이션 충돌쌍**(정규식 noise 없음)과
   사람이 쓴 gold_path에서만 도출한다. **KG 위상에서 요구를 만들지 않는다.** 위상(여러 길)은 모델이
   "유효한 길을 찾았는지"만 판정하지, "무엇이 필수인지"를 정하지 않는다. → 그래프 모델이 거저 점수받지 못한다.

조문 단위 정규화(항 무시)로 gold가 인용한 항과 모델이 닿은 항이 달라도 같은 법적 노드로 본다.

판정(요구관계 r별): **SATISFIED**(두 노드 도달 + 모델이 그 쌍을 실제 경로로 이음=옳은 길) /
**COINCIDENT**(두 노드는 닿았으나 잇는 경로 없음=*맞는 자리, 틀린/없는 이유*) / **MISSED**(노드 미도달).

## 29. 결과 — origin 분리(가이드 §6 by-construction 방어)

44/80문항이 요구관계(R_gold≠∅) 보유. 나머지 36(단일·일반·무응답)은 *연결 불필요* → 분모서 제외(정직).

| 집합 | n | Path Recall | Coincidence | Conn Precision | F1 |
|------|---|------------|-------------|----------------|----|
| 전체 | 44 | 0.693 | 0.03 | 0.984 | 0.813 |
| generated(앵커 위) | 26 | 0.731 | 0.0 | 0.975 | 0.836 |
| **held_out(앵커 밖)** | 18 | **0.639** | 0.077 | 1.0 | 0.78 |
| ┗ held_out hop2 | 15 | 0.667 | 0.091 | 1.0 | 0.8 |
| ┗ **held_out hop3** | **3** | **0.5** | 0.0 | 1.0 | 0.667 |

- **핵심 방어**: **앵커 밖 held_out에서도 Path Recall 0.639** — 그래프가 *생성 때 심어둔 길*만 밟는 게
  아니라, 사람이 앵커 밖에서 쓴 문항의 필수 연결도 절반 이상 실제로 traverse한다. by-construction이 아닌
  **구조적 추론경로**의 증거. (generated 0.731과의 격차 0.09 = 앵커 프라이밍 효과의 상한.)

## 30. 충실도 — "맞는 *이유*로 맞았나"

노드를 완전히 도달한 30문항 중 **경로까지 완전(Path Recall=1)한 문항 = 29 → 0.967.**

- 즉 **그래프가 올바른 근거에 닿았을 때, 96.7%는 그 근거들을 실제 유효 경로로 *연결*해서 닿았다**(우연
  착지가 아니라 옳은 길). 이는 로드맵4 via-귀인("graph_only 51.7%")을 *경로 수준*에서 재확인.
- 유일한 예외 **#26**(용도상이, held_out): 요구쌍이 `고보령_제12조(규모판정) ↔ 고보령_제3조(자격)`로
  **동일 문서 내** 관계다. 개념 브리지는 *교차 문서 전용*이고 동일 문서 내 CONFLICTS_WITH도 없다 →
  **그래프에 이을 엣지 자체가 없어** 구조적으로 COINCIDENT. → **정직한 발견: 그래프의 길은 문서 *간*
  용도상이엔 강하나 문서 *내* 용도 구분엔 없다.** (이건 모델 결함이 아니라 KG 스키마의 경계.)

## 31. 어떤 길로 이었나 (동치류 실측) + 정밀도

| SATISFIED 엣지유형 | 건수 | 의미 |
|--------------------|------|------|
| CONFLICTS_WITH(직접) | 36 | 전문가 앵커 직접 |
| 개념브리지(DEFINES→Concept←DEFINES) | 6 | 충돌셋 밖을 개념으로 연결 |

- **두 종류의 길이 모두 실측**됨 → "경로는 하나가 아니다"라는 전제가 데이터로 확인. 참조사슬로만
  이은 충돌쌍은 0건(앵커 쌍이 개념-정착이라 자연스러운 결과).
- **Conn Precision 0.984**(held_out 1.0): 모델이 *gold끼리* 이은 연결은 거의 전부 실제 필수관계 →
  gold 근거를 엉뚱하게 잘못 잇는 *관계 환각*은 거의 없다. (비-gold 확장 noise는 별도 `spurious_links`로 기록.)

## 32. 메트릭이 잡아낸 진단(예시)

- **#23**(단일정밀/held_out): 요구쌍 `조특령§26의8 →(위임)→ 조특법§29의8`을 **MISSED**. 법-시행령 위임
  조문을 retriever가 끌어오지 못함을 *경로 수준에서* 포착 → 검색 개선 타깃(참조 확장)이 문항별로 특정됨.
- **#27**(용도상이): 작성 gold_path가 실재하지 않는 placeholder 노드(`지침_피보험자수`)를 참조 →
  R_gold 도출 시 **양끝이 해소된 gold 청크일 때만 채택**하는 가드로 phantom 쌍 오염 차단(→ PathRecall 정정).

## 33. 한계 (정직)

- **로깅 경로 = 검색단계 traverse(3개 엣지유형)만 포착** → Path Recall은 *하한*. Reasoner의 내부
  추론사슬(LLM chain-of-thought)은 측정 대상이 아니다.
- **Naive RAG는 그래프 traverse가 없어 Path Recall=0, Coincidence=1.0(정의상).** 따라서 이 지표의
  G vs N 비교는 "Naive엔 길이 없다"의 동어반복 — 그래서 **헤드라인은 G의 held_out Path Recall(0.639)과
  충실도(0.967)** 로 잡았다(그래프-내부 추론 충실도 진단). 노드 도달은 Context Recall로 따로 비교됨.
- **held_out hop3 n=3** — 일반화×멀티홉의 통계력 부족. 로드맵8(다중실행 CI)·추가 hop3 held_out 수집 전까지
  hop3 결론은 *방향성*으로만.
